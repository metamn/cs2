reboot

reboot is an interesting startup designing and developing social contracts worldwide.

when redesigning their old website they've migrated also their Wordpress backend to support their rich content structures.

{% include services.html %}

content design

content design

beside blog posts Reboot is dealing with complex case studies, team profile pages, work examples, multi-page mission statements — custom content Wordpress is not supporting by default.

in the old website this metadata was stored into various, non-interconnected, individual plugins which made content administration a pain.

as a first task the content structure was redesigned to be Wordpress native by eliminating all plugins.

simple administration

simple administration

we eliminated plugins by storing metadata in the default Wordpress database. this way anytime Wordpress is updated the data goes automatically with the update.

when metadata is integrated with Wordpress defaults the Dashboard is also cleaned up. Instead of having many extensions the new version had just one.

plugin development

a single plugin was created to facilitate the input of metadata on all these screens where the default Wordpress functionality was not enough.

using native controls and integration into the original workflow Reboot editors could naturally add new content without extra visual and functional overhead.
