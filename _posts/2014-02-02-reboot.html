reboot

reboot is an interesting startup designing and developing social contracts worldwide.

when redesigning their old website they've migrated also their Wordpress backend to support their rich content structures.

{% include services.html %}

content design

{% assign image_index = 0 %} {% include responsive-images.html type='from post' link="" %}

beside usual blog posts Reboot is dealing with complex case studies, team profile pages, work examples, multi-page mission statements — custom content Wordpress is not prepared yet.

these metadata was stored into various, non-interconnected, individual plugins which made content administration a pain.

as a first task this extended content structure was redesigned to be Wordpress native by eliminating all plugins.

simple administration

{% assign image_index = 1 %} {% include responsive-images.html type='from post' link="" %}

instead of content scattered through different administration pages we've created a single admin panel to manage all metadata.

in plus, all metadata was stored into the standard Wordpress database making it future proof. Anytime Wordpress is updated the data goes automatically with the update.

plugin development

{% assign image_index = 2 %} {% include responsive-images.html type='from post' link="" %}

a single plugin was created to facilitate the input of metadata on all these screens where the default Wordpress functionality was not enough.

using native controls and integration into the original workflow Reboot editors could naturally add new content without extra visual and functional overhead.

{% assign image_index = 3 %} {% include responsive-images.html type='from post' link="" %}

theme development

{% assign image_index = 4 %} {% include responsive-images.html type='from post' link="" %}

once data was stored and made user-friendly & future proof the last step was to create a Wordpress theme to showcase how this metadata can be displayed using Wordpress functions.

together with plugin usage a semantic HTML5 template was created to found the upcomming front-end development tasks for the new design.
